# Projeto de Treinamento em Qualidade de Software

Este repositório documenta um projeto prático de **Gestão de Testes** realizado como treinamento, simulando o ciclo de vida de qualidade de software em um ambiente controlado.

## 📋 Sobre o Projeto

Treinamento completo de QA utilizando boas práticas de gestão de testes, desde o planejamento até a execução, em um fluxo profissional de trabalho.

**Ambiente de teste:** [Swag Labs](https://www.saucedemo.com/) (aplicação demo para e-commerce)

## 🎯 Objetivos do Treinamento

- Aplicar gestão de projetos de teste com metodologias ágeis
- Estruturar artefatos de qualidade (épicos, histórias, casos de teste)
- Utilizar ferramentas profissionais do mercado
- Praticar documentação e rastreabilidade de testes

## 🛠️ Ferramentas Utilizadas

| Ferramenta | Finalidade |
|------------|------------|
| **Jira** | Gerenciamento de projetos e tarefas QA |
| **Zephyr Scale** | Criação e organização de casos de teste, ciclos e planos |
| **XMind** | Elaboração de mapas mentais para histórias de usuário |
| **Swag Labs** | Aplicação alvo para execução dos testes |

## 📊 Artefatos Produzidos

### Jira - Gestão do Projeto

- **Épico:** Funcionalidades principais do Swag Labs (login, catálogo, carrinho, checkout)
- **Histórias de Usuário:** Exemplo: *"Como usuário desejo filtrar e ordenar produtos por preço"*
- **Tarefas QA:** Planejamento, execução e relatórios de teste

### Zephyr Scale - Gestão de Testes

- **Casos de Teste:** Passo a passo documentado com pré-condições, dados e resultados esperados
- **Ciclos de Teste:** Organização por funcionalidade (ex: Ciclo 1 - Login)
- **Planos de Teste:** Estratégia, escopo e cronograma definidos

### XMind - Modelagem Visual

- Mapa mental da história do usuário, mostrando:
  - Atores envolvidos
  - Fluxos de ação (filtro e ordenação)
  - Elementos de interface
  - Regras de negócio
  - Resultados esperados
